Job Scheduler

Problem
Background jobs need to run reliably, handle retries, and avoid duplicated execution.

Decision
I implemented a simple worker loop using Go channels instead of cron-based scheduling.

Trade-offs
This approach is easy to reason about and test, but it requires the process to be always running and is not suitable for distributed scheduling without coordination.

Tech
Go, goroutines, channels
